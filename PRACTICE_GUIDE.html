Before You Start

Prerequisites

Make sure all of these are ready before starting the exercises.

1. AWS CLI configured

bash
# Verify AWS CLI is installed and configured
aws --version
aws sts get-caller-identity   # should return your account ID

2. CodeCommit HTTPS credential helper

bash
# Add to ~/.gitconfig  (run once)
git config --global credential.helper \
  '!aws codecommit credential-helper $@'
git config --global credential.UseHttpPath true
Alternative: SSH keys

You can also use SSH keys uploaded to IAM → Security credentials → SSH keys for CodeCommit. HTTPS credential helper is simpler for most lab setups.

3. IAM permissions needed

json
{
  "Effect": "Allow",
  "Action": [
    "codecommit:*"      // full access for lab purposes
  ],
  "Resource": "*"
}
Lab vs Production

codecommit:* is fine for a personal practice lab. In production, scope permissions to specific repositories and actions.

Project Structure

The TaskFlow API dummy project gives you a realistic multi-file Python project to work with. It contains real code, real tests, intentional bugs, and a Dockerfile.

codecommit-practice/ ├── app/ │ ├── app.py ← Flask REST API (GET/POST/PUT/DELETE /tasks) │ ├── config.py ← Env-based configuration │ ├── utils.py ← Validators, paginator, timestamp helpers │ └── notifications.py ← 3 intentional bugs for Ex 3 ├── tests/ │ └── test_app.py ← 20+ pytest tests ├── Dockerfile ├── requirements.txt ├── CHANGELOG.md └── .gitignore

The 3 bugs in notifications.py (for Exercise 3)

python
# BUG 1 (line ~28): wrong log level passed
def notify_task_created(self, task_id):
    return self._log(level="debug", ...)   # ← should be "info"

# BUG 2 (line ~33): missing return statement
def notify_task_completed(self, task_id):
    self._log(...)                          # ← caller gets None

# BUG 3 (line ~39): typo in dict key
entry = {"level": "warn", "messge": ...}  # ← should be "message"

bash

  --repository-name taskflow-api \
  --repository-description "TaskFlow API - CodeCommit practice" \

# Note the cloneUrlHttp from the output
2
Verify the repo was created
bash
aws codecommit get-repository \
  --repository-name taskflow-api

3
Clone the empty repository
bash
git clone \
  https://git-codecommit.ap-south-1.amazonaws.com/v1/repos/taskflow-api

Exercise 01

Clone & Commit

The most fundamental CodeCommit workflow: edit a file locally, commit it, push it, and verify the change in the AWS Console.

Task: Update the app version in config.py

bash
# 1. Make a visible change
sed -i 's/version": "1.0.0"/version": "1.0.1"/' app/app.py

# Also update CHANGELOG.md
echo "## [1.0.1] - $(date +%Y-%m-%d)" >> CHANGELOG.md
echo "### Changed" >> CHANGELOG.md
echo "- Bumped version to 1.0.1" >> CHANGELOG.md

# 2. Stage specific files (not git add .)
git add app/app.py CHANGELOG.md

# 3. Check what's staged
git diff --cached

# 4. Commit with a meaningful message
git commit -m "chore: bump version to 1.0.1"

# 5. Push to CodeCommit
git push origin main

# 6. Verify via CLI
aws codecommit get-branch \
  --repository-name taskflow-api \
  --branch-name main
What to check in AWS Console

Go to CodeCommit → taskflow-api → Commits. You should see your new commit at the top with the message "chore: bump version to 1.0.1". Click it to see the diff.

Bonus: View commit details via CLI

bash
# Get the latest commit ID
COMMIT_ID=$(aws codecommit get-branch \
  --repository-name taskflow-api \
  --branch-name main \
  --query 'branch.commitId' \
  --output text)

# Get full commit details
aws codecommit get-commit \
  --repository-name taskflow-api \
  --commit-id $COMMIT_ID
Exercise 02

Feature Branch

Create a feature branch, add a new endpoint, push the branch to CodeCommit, and verify the branch appears independently of main.

bash
# 1. Create and switch to a new branch
git checkout -b feature/add-search

# 2. Add a search endpoint to app/app.py
#    Open app/app.py and add this block before "if __name__":
python
# Add this new endpoint to app/app.py
@app.route("/tasks/search", methods=["GET"])
def search_tasks():
    """Search tasks by keyword in title or description."""
    query = request.args.get("q", "").lower()
    if not query:
        return jsonify({"error": "q parameter is required"}), 400
    results = [
        t for t in TASKS.values()
        if query in t["title"].lower() or query in t["description"].lower()
    ]
    return jsonify({"results": results, "count": len(results)})
bash
# 3. Stage and commit

git commit -m "feat: add GET /tasks/search endpoint"

# 4. Push the branch to CodeCommit
git push origin feature/add-search

# 5. Verify the branch exists in CodeCommit
aws codecommit list-branches \
  --repository-name taskflow-api

# 6. Compare branches via CLI
aws codecommit get-differences \
  --repository-name taskflow-api \
  --before-commit-specifier main \
  --after-commit-specifier feature/add-search
Exercise 03

Fix Bugs on a Hotfix Branch

The file app/notifications.py has 3 intentional bugs. Create a hotfix branch from main, fix all 3, and push.

The 3 bugs to find and fix

Bug 1 — notify_task_created: level should be "info", not "debug"
Bug 2 — notify_task_completed: missing return before self._log(...)
Bug 3 — notify_task_deleted: dict key is "messge", should be "message"

bash
# 1. Switch back to main first (hotfix branches from main)
git checkout main
git pull origin main

# 2. Create hotfix branch
git checkout -b hotfix/fix-notifications

# 3. Open app/notifications.py and fix all 3 bugs
#    (see the bug locations in the file comments)

# 4. Commit each fix separately for clear history
git add app/notifications.py
git commit -m "fix: use 'info' level in notify_task_created"

# (make your fix 2 edit, then:)
git add app/notifications.py
git commit -m "fix: add missing return in notify_task_completed"

# (make your fix 3 edit, then:)
git add app/notifications.py
git commit -m "fix: correct typo 'messge' → 'message' in notify_task_deleted"

# 5. Push the hotfix branch
git push origin hotfix/fix-notifications

# 6. See commit log for this branch
git log --oneline --graph origin/main..hotfix/fix-notifications
Exercise 04

Pull Requests

Create, review, and merge a Pull Request entirely via the AWS CLI — then also try it in the Console.

bash
# 1. Create a Pull Request from hotfix → main
aws codecommit create-pull-request \
  --title "fix: resolve 3 bugs in notifications.py" \
  --description "Fixes incorrect log level, missing return, and typo in Notifier class" \
  --targets repositoryName=taskflow-api,sourceReference=hotfix/fix-notifications,destinationReference=main

# Note the pullRequestId from output (e.g. "1")

# 2. List all open PRs
aws codecommit list-pull-requests \
  --repository-name taskflow-api \
  --pull-request-status OPEN

# 3. Get PR details
aws codecommit get-pull-request \
  --pull-request-id 1

# 4. Post a review comment on the PR
aws codecommit post-comment-for-pull-request \
  --pull-request-id 1 \
  --repository-name taskflow-api \
  --before-commit-id $(git rev-parse origin/main) \
  --after-commit-id  $(git rev-parse origin/hotfix/fix-notifications) \
  --content "LGTM! All 3 bugs fixed correctly."

# 5. Approve the PR
aws codecommit update-pull-request-approval-state \
  --pull-request-id 1 \
  --revision-id $(aws codecommit get-pull-request --pull-request-id 1 \
    --query 'pullRequest.revisionId' --output text) \
  --approval-state APPROVE

# 6. Merge the PR (squash merge)
aws codecommit merge-pull-request-by-squash \
  --pull-request-id 1 \
  --repository-name taskflow-api \
  --source-commit-specifier hotfix/fix-notifications \
  --destination-commit-specifier main \
  --commit-message "fix: resolve 3 notification bugs (#1)"

# 7. Pull merged changes locally
git checkout main && git pull origin main
Three merge strategies available

merge-pull-request-by-fast-forward — no merge commit, linear history
merge-pull-request-by-squash — squashes all commits into one
merge-pull-request-by-three-way — explicit merge commit (GitHub-style)

Exercise 05

Create & Resolve a Merge Conflict

Two developers edit the same line on different branches. Practice resolving the conflict before merging.

bash
## ── Branch A: change MAX_TASKS to 500 ──────────────────────────
git checkout main && git pull origin main
git checkout -b conflict/branch-a

# Edit app/config.py line:  MAX_TASKS = int(os.getenv("MAX_TASKS", 1000))
# Change 1000  →  500
sed -i 's/MAX_TASKS", 1000/MAX_TASKS", 500/' app/config.py
git add app/config.py
git commit -m "config: reduce MAX_TASKS to 500"
git push origin conflict/branch-a

## ── Branch B: change MAX_TASKS to 2000 ─────────────────────────
git checkout main
git checkout -b conflict/branch-b

# Edit the SAME line in app/config.py: change 1000  →  2000
sed -i 's/MAX_TASKS", 1000/MAX_TASKS", 2000/' app/config.py
git add app/config.py
git commit -m "config: increase MAX_TASKS to 2000"
git push origin conflict/branch-b

## ── Merge Branch A into main first (succeeds) ───────────────────
git checkout main
git merge conflict/branch-a
git push origin main

## ── Try to merge Branch B (this will CONFLICT) ──────────────────
git merge conflict/branch-b
# Git will report: CONFLICT (content): Merge conflict in app/config.py

## ── Open app/config.py, find the conflict markers:
# <<<<<<< HEAD
# MAX_TASKS = int(os.getenv("MAX_TASKS", 500))
# =======
# MAX_TASKS = int(os.getenv("MAX_TASKS", 2000))
# >>>>>>> conflict/branch-b

## ── Decide: keep 2000, remove conflict markers ───────────────────
git add app/config.py
git commit -m "merge: resolve MAX_TASKS conflict, keep 2000"
git push origin main

Tags & Releases

Tag a stable commit as a release version and explore tags via CLI.

bash

git tag -a v1.0.0 -m "Release v1.0.0 - Initial stable release"

git push origin --tags

# 4. List tags via CLI
aws codecommit list-tags-for-resource \
  --resource-arn $(aws codecommit get-repository \
    --repository-name taskflow-api \
    --query 'repositoryMetadata.Arn' --output text)

# 5. Get commit behind a tag
git show v1.0.0 --stat

# 6. Create a patch release after another fix
git tag -a v1.0.1 -m "Release v1.0.1 - Bug fixes"
git push origin v1.0.1

# 7. Checkout code at a specific tag (detached HEAD)
git checkout v1.0.0
git checkout main   # return to main
Exercise 07

SNS Notifications / Triggers

Set up a CodeCommit trigger that sends a notification to an SNS topic on every push to main. This is the foundation for CI/CD integrations.

bash
# 1. Create an SNS topic
TOPIC_ARN=$(aws sns create-topic \
  --name taskflow-codecommit-events \
  --query TopicArn --output text)

echo "Topic ARN: $TOPIC_ARN"

# 2. Subscribe your email to the topic
aws sns subscribe \
  --topic-arn $TOPIC_ARN \
  --protocol email \
  --notification-endpoint [email protected]
# Check your email and confirm the subscription

# 3. Grant CodeCommit permission to publish to the topic
aws sns set-topic-attributes \
  --topic-arn $TOPIC_ARN \
  --attribute-name Policy \
  --attribute-value '{
    "Version":"2012-10-17",
    "Statement":[{
      "Effect":"Allow",
      "Principal":{"Service":"codecommit.amazonaws.com"},
      "Action":"sns:Publish",
      "Resource":"'$TOPIC_ARN'"
    }]
  }'

# 4. Create the trigger on the repository
aws codecommit put-repository-triggers \
  --repository-name taskflow-api \
  --triggers '[{
    "name": "push-to-main",
    "destinationArn": "'$TOPIC_ARN'",
    "branches": ["main"],
    "events": ["updateReference"],
    "customData": "taskflow-push"
  }]'

# 5. Verify triggers are set
aws codecommit get-repository-triggers \
  --repository-name taskflow-api

# 6. Test: make a commit and push to main — check your email!
echo "# Trigger test $(date)" >> CHANGELOG.md
git add CHANGELOG.md
git commit -m "test: trigger notification"
git push origin main
Exercise 08

Inspect History & Diff

Practice reading commit history, diffing commits, and viewing file blame — all via the CLI.

bash
# 1. List recent commits on main
aws codecommit get-differences \
  --repository-name taskflow-api \
  --after-commit-specifier main

# 2. Get commits between two points
aws codecommit get-commit-history \
  --repository-name taskflow-api \
  --commit-id main

# 3. Diff two specific commits
BEFORE=$(git rev-parse HEAD~2)
AFTER=$(git rev-parse HEAD)

aws codecommit get-differences \
  --repository-name taskflow-api \
  --before-commit-specifier $BEFORE \
  --after-commit-specifier $AFTER

# 4. Get the content of a specific file at a commit
aws codecommit get-file \
  --repository-name taskflow-api \
  --commit-specifier main \
  --file-path app/app.py \
  --query fileContent \
  --output text | base64 --decode

# 5. View the file tree at a commit
aws codecommit get-folder \
  --repository-name taskflow-api \
  --commit-specifier main \
  --folder-path /

# 6. View blame (who changed each line)
git blame app/app.py

# 7. Search commits by message
git log --oneline --grep="fix:"

# 8. See which files changed in the last 3 commits
git log --name-only --oneline -3
